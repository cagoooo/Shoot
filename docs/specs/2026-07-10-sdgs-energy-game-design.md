## 1. 產品摘要

《地球守護隊：能量大作戰》是一款結合 SDGs、自然科學、工程設計、數學與環境教育的瀏覽器 3D 第一人稱遊戲。

玩家扮演「地球小隊員」，在基地選擇能源、組裝科幻能量工具，再進入受污染與資源失衡影響的區域，完成探索、淨化、回收、修復、調查及撤離任務。遊戲保留能量槍、強力手炮及多元工具的射擊樂趣，但不模擬真實槍械製作，不呈現血腥、肢解、屍體或寫實傷害。

完整版本以聯合國 17 項永續發展目標為長期內容藍圖。首個垂直切片〈垃圾風暴救援行動〉聚焦 SDG 7、12、13，用一節課可完成的流程驗證遊戲性、閱讀理解、跨載具操作與教學價值。

## 2. 產品目標

### 2.1 學生體驗目標

- 學生能在瀏覽器直接開始遊戲，不需安裝桌面程式。
- 學生能享受探索、組裝能量工具、對抗搗蛋怪及安全撤離的樂趣。
- 學生能看見零件選擇帶來的實際效果，而不是只背誦知識。
- 學生能理解「威力最高」不等於「最適合任務」。
- 學生能從行動理解 SDGs，再學習正式名稱與意義。
- 不熟悉第一人稱操作的學生也能透過輔助模式參與。
- 遊戲不以延長使用時間或增加黏著度為成功標準。

### 2.2 教學目標

- 以自然科學與工程設計循環為主軸。
- 將數學計算、資源配置、閱讀理解及環境教育融入任務。
- 讓學生經歷觀察、預測、選擇、測試、記錄、比較及改良。
- 讓學生理解永續問題同時包含環境、社會與經濟面向。
- 提供可在一節約 40 分鐘課堂中使用的教學流程。
- 以學生的選擇、證據及改良歷程作為學習成果，不以擊敗數量評量。

### 2.3 產品發展目標

- 首發完成一個 12–18 分鐘、高品質、可重玩的垂直切片。
- 從第一天預留電腦、觸控、控制器與未來多人合作架構。
- 使用資料驅動方式增加零件、敵人、任務與 SDGs 章節。
- 先完成單人 PvE，日後再加入 2–4 人合作模式。
- 首發不要求學生註冊；教師端與雲端資料採分階段建置。
- 所有首發功能都必須能建置為靜態網站並部署到 GitHub Pages。
- BGM 可由 Suno 輔助製作，但學生端只播放已審核的本地音檔，不直接連接生成服務。

## 3. 設計原則

### 3.1 先好玩，再從行動理解

教育內容直接影響任務結果。學生先做選擇、觀察結果，再由學習卡連結正式概念。遊戲不使用大量選擇題阻斷探索與戰鬥。

### 3.2 先白話，後知識

- 學生介面使用生活化名稱、圖示、顏色及一句話說明。
- 「為什麼？」知識卡提供正式科學概念與 SDGs 對應。
- 教師報告保留能源效率、材料再利用等正式用語。
- 主要按鈕盡量控制在二至四個字。
- 每段任務說明最多兩個短句。
- 重要文字提供語音朗讀。

### 3.3 非血腥、非污名化

- 可射擊敵人只代表污染、浪費、失控機械或虛構能量。
- 敵人不代表任何國家、職業、性別、族群、疾病或弱勢者。
- 貧窮、飢餓、不平等、健康與和平議題不以射擊解決。
- 不鼓勵瞄準頭部；弱點設在發光核心、污染罐或故障零件。
- 失敗描述為防護罩耗盡並由救援小飛機回收，不描述死亡。

### 3.4 多種解法

學生可以淨化、固定、修復、引導或避開威脅。任務不把單一零件或方案標成唯一正解，結算畫面說明不同方案的結果與代價。

### 3.5 跨載具公平

輔助瞄準、自動拾取、圖像提示及簡化操作不降低徽章或學習成果。學生不因設備、反應速度或遊戲經驗不同而受到較低評價。

## 4. 世界觀與角色定位

地球部分地區因垃圾、污染、浪費能源、極端天氣與棲地破壞，產生神祕的「搗蛋能量」。搗蛋能量會讓廢棄物、機器和植物成為具有卡通殭屍動作的「搗蛋怪」。牠們不是人類病患，也不是需要被殺害的生物。

玩家加入「地球守護隊」，成為地球小隊員，使用能量槍、強力手炮、回收泡泡、修理工具及其他永續裝置：

- 清理污染。
- 回收材料。
- 修復城市設備。
- 保護動植物與棲地。
- 節省能源。
- 調查問題來源。
- 讓搗蛋怪與失控機械恢復正常。

故事的核心觀念是：單一技術看似環保，不代表整個系統必然永續。學生必須同時考慮能源、材料、生態、居民需求、安全、公平與長期影響。

## 5. 核心遊戲循環

一局標準任務為 12–18 分鐘：

1. 接收 SDGs 任務。
2. 調查天氣、地圖與區域問題。
3. 提出工具與能源選擇的預測。
4. 在基地組裝及試用能量工具。
5. 進入區域探索、採集、淨化與修復。
6. 選擇繼續探索或安全撤離。
7. 查看永續影響與學習回顧。
8. 根據證據改良設計，再次挑戰。

### 5.1 課堂時間建議

| 時間 | 活動 |
| --- | --- |
| 5 分鐘 | 教師說明情境，學生觀察問題 |
| 5 分鐘 | 學生提出預測並選擇工具 |
| 12–18 分鐘 | 探索、解題、淨化與撤離 |
| 5 分鐘 | 查看永續行動卡 |
| 5–10 分鐘 | 比較方案、分享理由及討論 SDGs |

### 5.2 任務長度發展

- 首發：12–18 分鐘標準任務。
- 後續：5–8 分鐘實驗任務。
- 後續：25–30 分鐘大型綜合任務。
- 所有任務支援暫停、自動存檔及稍後繼續。

### 6.1 任務類型

- 清理與守護：能量槍、手炮、泡泡及防護工具。
- 修復與建造：修設備、組裝工具、恢復環境。
- 調查與選擇：找證據、比較資料、判斷長期影響。
- 幫助與合作：分配資源、幫助角色、公平合作。

### 6.2 章節結構

1. 認識生活中的問題。
2. 完成行動任務。
3. 使用不同工具重新挑戰。
4. 解鎖正式 SDG 名稱及白話說明。
5. 取得一項可在學校或家庭實踐的生活行動。

## 8. 搗蛋怪與非血腥戰鬥

### 8.1 基本敵人

| 名稱 | 行為 | 代表問題 | 處理方式 |
| --- | --- | --- | --- |
| 垃圾黏黏怪 | 緩慢前進並留下垃圾 | 廢棄物過多 | 固定、分類、回收 |
| 偷電小怪 | 吸走設備電力 | 能源浪費 | 找核心、切斷偷電線 |
| 黑煙噴噴怪 | 製造黑煙降低視線 | 空氣污染 | 關污染罐、開清理器 |
| 汙水泡泡怪 | 污染水池及管線 | 水污染 | 淨水、修復漏水處 |
| 亂長藤蔓怪 | 阻塞道路 | 棲地失衡 | 辨認異常藤蔓、保護原生植物 |
| 壞掉機器人 | 重複搬運及丟棄物資 | 生產流程浪費 | 找出錯誤步驟、重新設定 |
| 假消息精靈 | 製造真假混合提示 | 錯誤資訊 | 蒐集證據、比對來源 |
| 暴風小旋風 | 推動玩家與場景物品 | 極端天氣 | 找掩護、修防災設備 |

假消息精靈不能用能量槍處理；找到足夠證據後才會恢復正確資訊。

### 8.2 戰鬥動作

- 淨化：清除搗蛋能量。
- 固定：使用回收泡泡暫時限制移動。
- 修復：關閉污染源或修好設備。
- 引導：將威脅引到回收區或遠離生態區。
- 避開：避免不必要的能源消耗。

### 8.3 行為提示

- 黃色閃光：準備攻擊。
- 紅色範圍：危險區域。
- 藍色圖示：可用工具處理。
- 綠色圖示：附近有需保護的生物或設施。
- 不使用背後貼臉生成或強烈跳躍驚嚇。

## 9. 失敗、撤離與難度

### 9.1 分級損失

- 中年級／學習模式：不遺失物品。
- 高年級標準模式：保留藍圖、研究成果及基本裝備，只損失部分當局材料；救援小飛機可帶回一件重要物品。
- 挑戰模式：提高撤離風險，但不影響主要教學進度。

### 9.2 中年級輔助模式

- 較強自動瞄準。
- 更長的危險提示。
- 限制同時攻擊的敵人數量。
- 圖像化任務引導與語音朗讀。
- 迷路時顯示地面箭頭。
- 較大的互動範圍。
- 自動拾取附近材料。
- 可隨時暫停及切換輔助設定。

## 10. 首個垂直切片：垃圾風暴救援行動

### 10.1 任務背景

「綠環回收站」因強烈暴雨造成控制系統故障。分類機開始亂丟垃圾、浪費電力並產生搗蛋能量。玩家必須救援維修小機器人、修正分類及能源設定、停止垃圾風暴並安全撤離。

主要連結：SDG 7、12、13。

### 10.2 任務流程

| 時間 | 階段 | 行動 | 學習內容 |
| ---: | --- | --- | --- |
| 0–2 分鐘 | 接收任務 | 查看天氣、地圖及回收站狀況 | 觀察問題、找重要資訊 |
| 2–4 分鐘 | 組裝工具 | 選能源盒、武器及輔助零件 | 預測能源、重量及散熱需求 |
| 4–7 分鐘 | 回收站入口 | 學移動、淨化、固定及回收 | 基本操作、垃圾分類 |
| 7–10 分鐘 | 分類大廳 | 修輸送帶、救援機器人 | 材料分類、流程順序 |
| 10–14 分鐘 | 垃圾風暴機 | 分三步停止頭目 | 能源效率、系統思考 |
| 14–16 分鐘 | 撤離 | 選擇優先攜帶內容 | 風險與資源選擇 |
| 16–18 分鐘 | 回顧 | 比較預測、結果及改良 | 證據、反思、改良 |

### 10.3 地圖區域

1. 守護隊準備室。
2. 回收站入口。
3. 垃圾堆放區。
4. 分類輸送大廳。
5. 能源控制室。
6. 垃圾風暴機。
7. 屋頂撤離點。
8. 可選維修小路：戰鬥較少，但需修復電路。

### 10.4 能源選擇

- 陽光能源盒：平常充電穩定，暴雨時補充速度較慢。
- 風力能源盒：風大時補充較快，風暴過強時會為安全暫停。
- 回收電池盒：不受天氣影響，容量較小，需尋找充電站。

### 10.5 頭目：垃圾風暴機

1. 觀察紙類、金屬、塑膠與不可回收物，停止錯誤輸送帶。
2. 清除三個搗蛋核心，可選淨化、固定或修好自動回收機。
3. 選擇快速全力、慢速省電或分區輪流運轉方案。

選錯分類不直接失敗。遊戲以白話提示學生重新觀察材料。

### 10.6 撤離選擇

玩家優先帶走一類物品：

- 可再次使用的珍貴零件。
- 回收站研究紀錄。
- 額外能源。
- 受損維修小機器人。

完成額外任務可增加帶回物品的數量。

### 10.7 行動徽章

- 省電高手。
- 回收達人。
- 修理能手。
- 安全守護者。
- 改良小博士。

每枚徽章都顯示取得理由，不只顯示分數。

## 11. 學生介面

### 11.1 啟動流程

1. 選擇中年級輔助或高年級標準模式。
2. 選擇第一次玩或已熟悉操作。
3. 開始任務。

不要求真實姓名、電子郵件或正式帳號。可選擇輸入教師指定的匿名代碼。

### 11.2 基地四區

- 今天任務：選地圖並查看 SDG 主題。
- 工具桌：組裝能量工具。
- 試玩區：測試力量、省電、距離與降溫。
- 我的行動紀錄：查看結果與改良歷程。

### 11.3 組裝畫面

- 支援拖曳，也支援點零件後再點位置。
- 綠色向上箭頭表示能力增加。
- 橘色向下箭頭表示能力降低。
- 「為什麼？」以一句白話解釋原因。
- 不顯示「錯誤配法」，只提醒可能遇到的困難。

### 11.4 第一人稱 HUD

- 中央：簡單準星。
- 左下：防護罩。
- 右下：工具能量與溫度。
- 上方：一行目前任務。
- 側邊：泡泡、修理及掃描工具。
- 過熱時以顏色、提示音及「先讓工具休息一下」提醒。

### 11.5 學習回顧

任務結束依序顯示：

1. 我做了什麼。
2. 發生了什麼。
3. 下次想怎麼改。

系統產生「我的永續行動卡」，可存成圖片、列印或在投影幕分享。

## 12. 無障礙與舒適性

- 介面文字可放大。
- 重要內容提供語音朗讀。
- 字幕可調整大小及背景。
- 顏色資訊同時搭配圖案或文字。
- 可關閉視角晃動、閃光及動態模糊。
- 可調整視角靈敏度與重新設定按鍵。
- 閱讀說明時停止倒數。
- 不使用高頻閃爍特效。
- 可隨時切換輔助模式，不影響徽章。
- 提供固定準星、較寬視野及快速轉身選項。

## 13. 跨載具操作與介面

### 13.1 統一輸入動作

遊戲規則只接收移動、轉向、主要工具、輔助工具、互動、切換、任務及暫停等共通動作。鍵盤滑鼠、觸控及控制器分別將實際輸入轉換成共通動作。

### 13.2 觸控配置

- 3D 任務以橫向畫面為主。
- 左側虛擬搖桿控制移動。
- 右半畫面滑動控制視角。
- 主要工具使用大型拇指按鈕。
- 互動按鈕只在接近目標時出現。
- 支援左右手模式、按鈕大小、透明度與位置微調。
- 任務、組裝及報告支援直向與橫向。

### 13.3 響應式規格

- 重要觸控按鈕至少約 48 CSS 像素。
- 避開瀏海、圓角及系統手勢安全區域。
- 小螢幕使用分頁，不縮小主要文字。
- 不依賴滑鼠懸停查看說明。
- 軟體鍵盤不得遮住輸入欄位。
- 裝置旋轉、分割畫面及視窗改變後自動重排。

### 13.4 行動效能模式

- 省電模式：目標 30 FPS，降低陰影、粒子、反射及遠景。
- 流暢模式：動態解析度，優先保持操作反應。
- 高畫質模式：效能測試通過後才提供。
- 降低效果時保持文字與字幕清晰。

## 14. 技術架構

### 14.1 技術選擇

- Babylon.js：3D 場景、攝影機、動畫、碰撞、粒子及音效。
- TypeScript：遊戲規則、任務、工具、敵人及學習紀錄。
- React：基地、工具組裝、設定、學習報告及教師介面。
- Vite：開發與建置。
- glTF／GLB：3D 資源。
- IndexedDB：本機進度。
- 靜態網站託管：首發部署。

WebGL 2 是基本渲染路徑；WebGPU 是支援設備的可選加速，不是遊玩必要條件。

### 14.2 模組邊界

- 顯示層：場景、角色、特效、音效。
- 遊戲規則層：能源、溫度、任務及敵人狀態。
- 內容資料層：SDGs、關卡、零件、對話及提示。
- 學習紀錄層：選擇、測試、改良及結果。
- 存檔層：本機資料與版本轉換。
- 輸入層：鍵盤滑鼠、觸控及控制器轉換。
- 連線層：首版僅定義介面，後續接入多人服務。

### 14.3 資料驅動

零件、武器、敵人、任務、徽章、白話說明、延伸知識及 SDGs 連結都由具版本的資料格式描述。新增內容不複製核心程式。

### 14.4 物理與射擊

- 能量槍使用快速射線判定。
- 泡泡、種子等慢速工具使用飛行物判定。
- 場景只對必要物品啟用物理。
- 垃圾分類使用簡化碰撞形狀。
- Havok 物理只在推動、堆疊或合作搬運確有需求後加入。

### 14.5 GitHub Pages 部署規格

GitHub Pages 是首發與校園試玩版的主要發布方式。學生透過公開網址進入，不需下載安裝檔。

- Vite 正式建置輸出到 `dist/`，GitHub Pages 只發布建置成果。
- 使用 GitHub Actions 執行檢查、建置及 Pages 部署，不使用 `gh-pages` 套件維護獨立部署分支。
- 合併或推送到 `main` 後，通過測試才自動部署。
- Vite 的 `base` 必須依 GitHub 儲存庫子路徑設定，禁止假設網站位於網域根目錄。
- 圖片、模型、音效、Web Worker、WASM 及動態載入內容都必須使用可套用 `base` 的資源路徑。
- 應用程式導航不依賴伺服器端改寫；首發採單頁狀態或 hash 路由，避免重新整理後出現 404。
- PWA Service Worker 的作用範圍、快取路徑及更新提示必須支援 GitHub Pages 子路徑。
- 每次部署產生可查詢的版本資訊，方便辨識快取是否仍為舊版。
- 發布流程保留自訂網域能力，但首發不以自訂網域為必要條件。

GitHub Pages 只能託管靜態前端，不能承擔教師資料庫、正式帳號或多人權威伺服器。未來加入這些功能時，遊戲前端仍可留在 GitHub Pages，後端則使用獨立且通過個資及安全審查的服務。

任何會放入瀏覽器程式的設定都視為公開資訊。私人金鑰、管理員憑證及伺服器密鑰不得寫入原始碼、Vite 環境變數或 GitHub Pages 建置成果。

## 15. 音樂、音效與 Suno 製作流程

### 15.1 使用方式

Suno 定位為阿凱老師在開發期間產生、挑選及整理遊戲 BGM 的音樂製作平台，不是提供學生操作的課程工具，也不是遊戲執行時依賴的線上服務。

- 由阿凱老師使用自己的成人 Suno 帳號產生並挑選適合遊戲的 BGM。
- 學生只在遊戲內聆聽完成並通過審核的音樂，不參與 Suno 登入、產生、挑選或下載流程。
- 使用付費 Pro 或 Premier 方案期間產生準備納入遊戲的音樂。
- 以原創純音樂提示詞產生候選曲，不模仿指定歌手、作曲家、遊戲或受保護作品。
- 不上傳沒有完整權利的歌曲、錄音、歌詞或聲音。
- 不使用複製真人聲音、兒童聲音或可辨識人物聲音的功能。
- 從 Suno 匯出 WAV 或可取得的最高品質檔案，再於本機完成剪輯、循環、音量及壓縮處理。
- 遊戲與 GitHub Pages 只包含已核准的成品，不包含 Suno 帳號權杖或服務憑證。

目前規格不依賴 Suno 公開 API。若未來 Suno 提供正式 API，必須重新審查條款、費用、隱私、金鑰保護、服務穩定性及兒童使用限制，核准後才可評估串接。不得使用非官方逆向 API。

### 15.2 授權與來源紀錄

每一首候選及正式 BGM 都建立資產紀錄：

- 曲目內部編號及學生版名稱。
- Suno 原始作品連結或識別碼。
- 產生日期及當時帳號方案。
- 原始提示詞與使用的模型。
- 原始下載檔及檔案雜湊。
- 是否使用延伸、重混、音訊上傳或分軌。
- 本機剪輯、混音及循環處理紀錄。
- 正式遊戲檔案名稱、長度、格式及版本。
- 產生當日的官方權利說明或條款存檔。
- 內容與授權審核人及審核日期。

免費方案產生的作品不得直接升級為正式遊戲資產。之後訂閱付費方案不視為取得先前免費作品的回溯商用權。其他 Suno 使用者公開的歌曲只能在另有明確遊戲使用授權並完成書面紀錄時採用；主要流程以阿凱老師自己的付費帳號產生的作品為準，避免權利來源不明。

Suno 授予的商用使用權不等同於保證作品能取得著作權保護。若未來要販售、授權、參加競賽或大規模公開發布，應再次確認當時條款與適用地區法律。

### 15.3 音樂方向

音樂以「明亮、好奇、勇敢、合作、守護地球」為主，不使用寫實恐怖、沉重軍事或過度刺激的聲音。

| 情境 | 音樂感受 | 功能 |
| --- | --- | --- |
| 標題與基地 | 溫暖、明亮、容易親近 | 建立地球守護隊主題旋律 |
| 工具組裝 | 輕巧、帶規律節奏 | 幫助思考，不干擾閱讀 |
| 區域探索 | 好奇、帶少量緊張 | 保持探索動機 |
| 搗蛋怪接近 | 節奏稍加快，不製造恐怖 | 清楚提示危險提升 |
| 頭目任務 | 有力量、分段明確 | 配合修復及解題階段 |
| 安全撤離 | 積極、時間感增加 | 提醒行動但不造成恐慌 |
| 任務成功 | 放鬆、具成就感 | 支援回顧與討論 |
| 學習報告 | 平靜、簡單 | 不干擾閱讀及語音朗讀 |

六大世界各有一組容易辨認的聲音色彩，但共享「地球守護隊」短旋律，使 17 項 SDGs 章節保持整體感。

### 15.4 動態 BGM

- 探索、緊張及頭目音樂使用相同速度與調性設計，方便平順交叉淡化。
- 敵人接近時增加節奏層，不突然切換成大音量音樂。
- 修復完成、保護生物或找到正確解法時加入短音樂提示。
- 開啟說明、語音朗讀或重要對話時自動降低 BGM 音量。
- 暫停、切換分頁及裝置鎖定時安全停止；回到遊戲後平順恢復。
- 第一次進入網站需由使用者點擊開始後才播放聲音，符合瀏覽器自動播放限制。

### 15.5 檔案與 GitHub Pages

- 保留無損原始檔作為製作母帶，不直接放入網站。
- 網頁版本輸出壓縮音訊，並提供主格式與相容備援格式。
- 短循環避免可聽見的接縫、爆音及音量跳動。
- 長音樂分批載入；先載入基地音樂，再背景載入任務音樂。
- 音訊路徑必須支援 GitHub Pages 儲存庫子路徑。
- PWA 快取需記錄音樂版本，更新曲目時能移除舊快取。
- BGM、音效及語音使用不同音量控制，並提供一鍵靜音。

### 15.6 音樂驗收

- 連續循環三次沒有明顯接縫或爆音。
- 不遮蓋語音、提示音及學生對話。
- 15–20 分鐘遊玩不造成明顯疲勞或過度緊張。
- 手機及平板切換 App 後能正確暫停與恢復。
- 關閉 BGM 後不影響任務完成或學習判斷。
- 每一個正式音檔都有完整來源與授權紀錄。
- 學生試玩確認音樂能傳達情境，且不被誤認為恐怖或戰爭遊戲。

### 15.7 備援方案

音樂系統透過資產編號載入，不把曲名或 Suno 連結寫死在任務規則中。若 Suno 條款、服務或授權條件改變，可用自製音樂、委託作曲或其他具有清楚遊戲授權的音樂庫替換，不需重寫關卡。

## 16. 效能與載入目標

- 一般學校電腦至少穩定 30 FPS。
- 較新設備目標 60 FPS。
- 自動偵測低、中、高畫質。
- 垂直切片同畫面約 12–20 隻搗蛋怪。

- 單人任務短暫斷線時不遺失當前進度。
- GitHub Pages 網址在儲存庫子路徑下可正確載入所有模型、音效及任務資料。

### 里程碑 6：六大世界代表關卡

每個世界至少完成一個代表任務，驗證環境與社會議題都能用合適玩法呈現。

### 里程碑 7：教師課堂功能

完成匿名課堂代碼、教師派發、班級統計及匿名報告。

### 里程碑 8：17 項 SDGs 完整版

每項 SDG 至少一個完整章節，並提供跨目標大型任務。

### 里程碑 9：2–4 人合作

建立正式連線架構，完成分工、分享資源、共同撤離及 SDG 17 合作任務。

## 23. 主要風險與控制

| 風險 | 控制方法 |
| --- | --- |
| 17 項 SDGs 範圍過大 | 先完成垂直切片，再按六大世界擴充 |
| 射擊蓋過教育 | 修復、回收、調查同樣影響成果 |
| 教育內容像考題 | 先行動、看結果，最後連結概念 |
| 名詞艱深 | 學生白話、延伸知識、教師正式用語三層設計 |
| 觸控操作困難 | 統一輸入層、專用 UI、真機測試 |
| 學校設備效能不足 | 動態畫質、物件池、分批載入、低畫質模式 |
| 第一人稱暈動 | 關晃動、固定準星、視角調整、輔助模式 |
| SDGs 過度簡化 | 教師及領域專家審閱、呈現多種解法 |
| 學生資料風險 | 首發不登入、本機保存、匿名及資料最小化 |
| 多人功能拖累首版 | 只預留介面，合作模式後期獨立開發 |
| AI 音樂授權或條款改變 | 只用付費期間自產作品、保存證據、維持可替換資產層 |
| 音樂干擾閱讀或造成緊張 | 純音樂、分軌音量、語音閃避、學生試聽 |
