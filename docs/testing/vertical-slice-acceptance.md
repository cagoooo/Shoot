# 0.1.0 垂直切片驗收紀錄

更新日期：2026-07-11

## 已通過範圍

- 啟動頁可選「中年級輔助」與「高年級標準」。
- 基地可前往工具桌、3D 試玩區、今日任務與學習紀錄。
- 〈垃圾風暴救援行動〉包含任務說明、工具準備、路線選擇、四類垃圾分類、三個核心淨化、能源方案、限量撤離與 SDGs 行動回顧。
- 主路與維修小路皆可完成；高年級完整流程另有獨立端到端測試。
- 檢查點可回復至安全階段；教材載入失敗與未預期錯誤都有學生看得懂的復原入口。
- 匿名錯誤紀錄不匯出錯誤原文、堆疊、個資或存檔內容。
- 桌機、平板與手機代表尺寸已有自動化響應式驗證。
- 效能監測器已接入實際 Babylon 渲染迴圈，降級時調整硬體縮放，而非只停留在規則測試。
- 已建立離線與未來線上共用的多人連線埠，以及可序列化的移動、工具、修理與互動指令；首版仍保持純單人離線。
- 完整任務設計時間為 12–18 分鐘；課堂實際時間仍待首輪學生試玩記錄。
- 內容採風格化科幻、淨化與修復表現，已檢查無血腥、屍體、真實槍械教學及跳躍驚嚇。
- 學習行動明確連結 SDG 7（能源方案）、SDG 12（分類與回收）及 SDG 13（降低垃圾風暴影響）。
- 本機存檔、檢查點恢復、匿名行動卡與匿名錯誤匯出皆有自動化測試；不含公開排名或學生個資。

## 效能預算

- 低 FPS 門檻：低於 28 FPS 持續 300 個取樣（約 5 秒／60 Hz）後降一級。
- 單次低效能事件：最多降一級；穩定 120 個取樣後才可視為新事件。
- 初始介面 JS 目標：壓縮前低於 300 KB；3D/Babylon 採延遲載入。
- 3D 共用延遲區塊目標：gzip 低於 250 KB；若正式美術加入後超標，需再拆分場景資源。
- 電腦可用品質基準為至少 30 FPS；自動降級的保護門檻設為持續低於 28 FPS。
