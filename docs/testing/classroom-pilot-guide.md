# 課堂試玩指引

適用版本：`0.1.0-vertical-slice`　　建議時間：一節課 35–40 分鐘

## 學習目標

學生能說明垃圾分類、能源選擇與安全撤離的理由，觀察不同工具模組造成的力量、耗能與降溫差異，並提出下一次更永續的改良想法。

## 課前準備（教師）

1. 用教師裝置先完整走一次〈垃圾風暴救援行動〉。
2. 確認每台載具的瀏覽器、聲音、鍵盤／滑鼠或觸控皆可用。
3. 優先安排兩人共用一台：一人操作、一人擔任「永續觀察員」，中途交換。
4. 中年級預設「中年級輔助」；高年級使用「高年級標準」。
5. 正式 BGM 未完成授權台帳前保持關閉或使用已稽核素材。

## 建議流程

| 時間 | 活動 | 引導問題 |
|---|---|---|
| 5 分鐘 | 認識基地與 SDG 7、12、13 | 能源、垃圾與氣候之間有什麼關係？ |
| 5–8 分鐘 | 工具桌及試玩區 | 換一個模組後，哪一項效果最明顯？ |
| 12–18 分鐘 | 垃圾風暴任務 | 為什麼選這條路、這個分類或能源方案？ |
| 8 分鐘 | 永續行動紀錄與同伴分享 | 下次要怎樣省電、減少垃圾或提高安全？ |

## 觀察紀錄

- 學生是否能獨立看懂主要按鈕與任務文字。
- 哪一段需要教師提示，提示後是否能繼續。
- 裝置型號、瀏覽器版本、是否橫向、聲音與平均 FPS。
- 是否出現暈眩、閃光不適、操作誤觸或卡關。
- 學生能否用自己的話解釋至少一個永續選擇。

不要記錄學生姓名、電子郵件、裝置識別碼或公開排名。若遇到錯誤，可先「重新載入目前階段」或「回基地」，需要回報時只匯出匿名錯誤紀錄。

## 課後討論

- 三種能源修復方法都能成功，為什麼結果仍不同？
- 如果學校也遇到垃圾與用電問題，我們能做哪一個小行動？
- 下一個 SDGs 任務想解決什麼問題？需要哪些觀察與證據？
